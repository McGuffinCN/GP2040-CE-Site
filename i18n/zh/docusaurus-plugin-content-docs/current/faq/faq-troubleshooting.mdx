## 输入问题

### **为什么我的按键顺序不对？**

可能有以下几种原因：

#### **1. Steam 中的映射不正确**

检查 Steam 中的控制器设置是否正确，包括全局设置和每个游戏的设置。有关解决此问题的更多信息，请参阅 [Steam 支持](https://partner.steamgames.com/doc/features/steam_controller/getting_started_for_players)。

#### **2. 输入模式不正确**

确认是否使用了正确的输入模式，不同的模式可能会更改输入映射（例如 Switch 模式与 XInput 模式，PS3/PS4 等）。有关更改输入模式的更多信息，请参阅 [GP2040-CE 使用指南](../usage.mdx#输入模式)。

#### **3. 按键接线错误**

检查您的按键是否根据控制器主板供应商提供的 GPIO 引脚映射正确连接到控制器主板。

#### **4. 固件不正确**

固件是基于特定 GPIO 引脚配置而构建的，如果固件的预期配置与您的配置不匹配，可能会导致问题。您可以在 [网页配置器](../web-configurator/menu-pages/02-gpio-pin-mapping.mdx) 中更新 GPIO 引脚映射，或者为您的设备安装正确的固件版本。

有关如何解决此问题的更多信息，请参阅下文的 [支持设备的固件错误](#1-支持设备的固件错误)。

### **按下按钮时为什么没有反应？**

1. 检查接线：按钮应有两个端子，一个端子连接到 GPIO 引脚，另一个连接到接地（GND）。通过短接这些线并验证是否收到输入信号来测试接线是否正常工作。
2. 检查按钮：按钮本身可能存在问题。尝试更换其他按钮。

### **为什么即使没有按下按钮，仍会收到输入？**

1. 检查接线：信号线可能无意中与地线（GND）或与 GND 相连的电线接触，从而导致短路。
2. 检查按钮：按钮本身可能损坏。尝试更换其他按钮。

### **上下或左右方向颠倒了，如何修复此问题？**

可能是您在控制器上启动了 [反转 Y 轴](../hotkeys.mdx#反转-y-轴) 或 [反转 X 轴](../hotkeys.mdx#反转-x-轴) 的快捷键功能。再次激活相应快捷键以关闭该功能。

<HotkeyNote />

### **我的 SOCD 覆盖模式更改了，该如何恢复为 X 覆盖模式？**

可能是您在控制器上激活了以下 SOCD 覆盖模式快捷键之一，再次激活快捷键可以关闭此功能：

- [SOCD 上优先](../hotkeys.mdx#socd-上优先)
- [SOCD 回中](../hotkeys.mdx#socd-回中)
- [SOCD 后输入优先](../hotkeys.mdx#socd-后输入优先)
- [SOCD 先输入优先](../hotkeys.mdx#socd-先输入优先)
- [SOCD 关闭](../hotkeys.mdx#socd-关闭)

<HotkeyNote />

### **为什么方向键会卡在输出方向到左/右摇杆？**

可能是您在控制器上启用了 [方向键左摇杆模式](../hotkeys.mdx#左摇杆模式) 或 [方向键右摇杆模式](../hotkeys.mdx#右摇杆模式) 的快捷键功能。再次激活对应快捷键可以关闭功能。

## 连接问题

### **为什么连接到 PC/主机的控制器无法识别？**

这可能有多种原因。检查以下问题：

#### **1. 键盘模式**

检查设备的输入模式是否设置为键盘模式。这会导致您的电脑仅识别控制器作为键盘，而不是游戏手柄。

#### **2. 网页配置器**

使用 网页配置器时，控制器会初始化为 RNDIS 设备，PC/主机会无法将其识别为控制器。检查控制器是否已启动到 网页配置器模式，并确保开机时未触发快捷键 <Hotkey buttons={["S2"]}/>。

### **为什么使用 USB Type-C 转 Type-C 无法连接到 PC/主机？**

由于设备硬件设计的原因，可能无法使用可逆的 USB Type-C。尝试翻转电缆方向。如果仍然不起作用，尝试使用 USB 2.0 Type-C 转 Type-C 电缆或带适配器的 USB 2.0 Type-C 转 Type-A 电缆。

### **为什么我的控制器在 PS4 或 PS5 上无响应？**

- **针对 PS4 的问题**：确认已启用 [PS4 输入模式](../web-configurator/menu-pages/01-settings.mdx#额外的-ps4-设置) 并上传密钥文件，或通过 USB 主机使用有效的认证设备。
- **针对 PS5 的问题**：确认已启用 [PS5 输入模式](../web-configurator/menu-pages/01-settings.mdx#额外的-ps5-设置)，并通过 USB 主机使用有效的认证设备。

有关更多信息，请参阅 [PS4/PS5 兼容性常见问题](../faq/faq-console-compatibility.mdx)。

### **为什么我无法访问 网页配置器？**

#### **1. 控制器未启动到 网页配置器模式**

如果 网页配置器页面超时，请确认控制器是否已启动到 网页配置器模式。

:::info 如何进入 Web 配置模式

- 插入设备时按住 <Hotkey buttons={["S2"]} />。
- 按住 <Hotkey buttons={["S2", "B3", "B4"]} /> 持续 5 秒。

:::

测试是否按下了正确的按键，例如使用 [Gamepad Tester](https://gamepad-tester.com/) 网站或其他应用（如 Steam 输入界面）进行验证。

#### **2. 网络中其他设备占用了 192.168.7.1 的 IPV4 地址**

如果网络中其他设备（例如路由器、交换机、IoT 设备等）占用了相同的 IP 地址，则会发生 IP 地址冲突，导致无法访问 网页配置器。

断开电脑与本地网络的连接（例如拔掉网线），然后重新尝试访问 网页配置器。

## 配置问题

### **为什么我的设备上的 PS4 或 PS5 输入模式无法正常工作？**

#### **1. 连接控制器后未按下 A1 键**

在连接已正确配置为 PS4 或 PS5 输入模式的控制器后，您需要按下快捷键 <Hotkey buttons={["A1"]} />，以便 PS4 或 PS5 识别控制器。操作后，游戏机应能正常接收控制器输入。

#### **2. 控制器模式配置错误**

确保在“设置”部分中已上传 PS4 的密钥文件或选择了 PS4/PS5 的 USB 主机模式。

有关更多信息，请参阅 [PS4 输入模式](../web-configurator/menu-pages/01-settings.mdx#额外的-ps4-设置) 或 [PS5 输入模式](../web-configurator/menu-pages/01-settings.mdx#额外的-ps5-设置) 文档。

#### **3. USB 主机端口未正确配置**

在 网页配置器的 [外围设备映射](../web-configurator/menu-pages/03-peripheral-mapping.mdx#usb-主机) 中，确认您的 USB 主机端口配置正确。不同设备制造商提供的 Data+ 和 Data- 的特定引脚可能会有所不同，请参考供应商的文档。

#### **4. 使用的控制器或适配器不兼容**

只有兼容的设备才能使 GP2040-CE 控制器通过 PS Passthrough 功能在 PS4 上正常工作，而并非所有设备都支持 PS5。

有关更多信息，请参阅 [PS4/PS5 兼容性常见问题](../faq/faq-console-compatibility.mdx)。

### **我无法通过快捷方式进入 网页配置器。如何手动进入 网页配置器？**

如果 **不擦除您的设备固件**，请按以下步骤操作：

1. 使用本 [强制 WebConfig 固件文件](/assets/force_webconfig.uf2) 并按照 [固件安装说明](../installation.mdx) 运行。  
2. 这会强制您的设备以 网页配置器模式进行启动。

### **为什么我的 RGB LED 显示的颜色不对？**

确认 `LED 格式` 设置正确，不同的 LED 类型使用的 RGB 数据顺序可能不同。

有关更多信息，请参阅 网页配置器菜单中的 [RGB LED 按键顺序](../web-configurator/menu-pages/05-led-configuration.mdx#rgb-led-按键顺序) 部分。

### **为什么我的 RGB LED 顺序不对？**

RGB LED 的顺序可以在 网页配置器中设置，正确的配置取决于硬件和用户如何布线 LED 链，或设备设计师如何在电路板上设置 LED 链。

有关更多信息，请参阅 网页配置器菜单中的 [RGB LED 按键顺序](../web-configurator/menu-pages/05-led-configuration.mdx#rgb-led-按键顺序) 部分。

### **更新设备后，为什么我的控制器停止工作了？**

更新设备后，输入设置突然异常，OLED 显示屏或 RGB LED 停止工作，很可能是因为您为设备安装了错误的固件，或者设备不受支持，因此 [下载页面](/downloads) 上的预编译版本可能无法正常运行。

#### **1. 支持设备的固件错误**

如果设备受支持，但安装了错误的固件，请按照 [固件安装说明](../installation.mdx) 操作并重新刷写正确的固件，确认固件文件名称与设备匹配。

#### **2. 不支持的设备**

如果您的设备不受支持，请联系您购买设备的供应商获取正确的固件文件，并按照 [固件安装说明](../installation.mdx) 操作。

如果供应商无法为设备提供支持，您需要自行研究并正确设置您的设备：

1. 从下载页面获取 **Pico GP2040 固件** 并按照 [固件安装说明](../installation.mdx) 操作刷入设备。
2. 不擦除设备固件，刷入 [强制 WebConfig 固件文件](/assets/force_webconfig.uf2)，强制设备以 网页配置器模式启动。
3. 进入 网页配置器，转到“配置 > GPIO 引脚映射”。
4. 对于每个游戏手柄输入，使用“按键映射功能”将输入映射到连接按钮的 GPIO 引脚，重复进行所有输入的映射操作。
5. 对于任何插件（如 RGB LED、OLED 显示屏、USB 主机端口等），需要确定其关联的 GPIO 引脚。这可以通过追踪 PCB 上的物理铜箔路径找到。

由于上述步骤完全取决于 PCB 制造商的设计，无法提供进一步的支持。
